Threat Notification

Windows Printer Spooler: Vulnerable to Privilege Escalation(CVE-2020-1048)

Vulnerability

Type: Vulnerability | Issue Date : 25-05-2020 | TLP: Green | Industry: All | Region: All  

Summary

On 12 May 2020, Windows-Internal researchers published a report about a privilege escalation vulnerability (CVE-2020-1048) presents in Windows Printer spooler of all versions. The print spooler is a software service that manages the printing processes. It accepts the print jobs from the users and makes sure that the resources are available. The print spooler executes with SYSTEM privileges and is network accessible. The vulnerability lies in the way the Windows print spooler service is handling the requests. It allows the users to arbitrarily writing to the file system.

 

An attacker would have to access the compromised machine and then with the help of specially crafted scripts or application can perform the elevation of privileges. The attacker has to provide the specially crafted input to spooler and the spooler service which is running with SYSTEM level privileges allows the arbitrary users to write to the file system. On successful exploitation the attacker can execute any random code with SYSTEM level privileges and then can perform any malicious activity on the compromised machine without any restriction. This vulnerability can be used to elevate the privileges, gain persistence, bypass EDR rules etc.

 

Impacted Products

·        Windows Printer Spooler.

Recommendations

Deloitte CTI recommends

·        Microsoft has released the patches with respect to the Operating Systems. The users should patch the vulnerability as the difficulty of exploitation of this vulnerability is very easy if the powershell is allowed to execute the commands.

·        Client side port check vulnerability - few detection parameters for the presence of any malicious file:

o   Any port that contains a file path ending with .exe or .dll extension is suspicious.

o   Powershell command -  “Get-PrinterPorts” could be used to perform a scan for any file-based ports.

o   Verify the registry entries at HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Ports

o   The attacker can simply use such commands for the exploitation - Add-PrinterPort -Name c:\windows\system32\ualapi.dll
