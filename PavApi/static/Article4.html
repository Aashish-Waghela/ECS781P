Threat Notification

Iranian APT group 'Chafer' targets travel industry and government entities in Kuwait and Saudi Arabia

Incident

Type: Incident | Issue Date : 27-05-2020 | TLP: Green | Industry: All | Region: All | Serial: G-TN-EN-01-13024

Summary

On May 21, 2020, security vendor BitDefender published a report detailing the Tactics, Techniques and Procedures (TTPs) of Iranian APT group 'Chafer' (a.k.a. APT39, Remix Kitten). According to the report, the group recently conducted two cyberespionage campaigns directed toward entities in Kuwait and Saudi Arabia. The threat group is known for targeting telecommunication and travel industries, as well as government entities in the Middle East, for the purpose of collecting personal information that serves Iran's geopolitical interests.

 

Researchers identified initial compromise based on reverse TCP shells and PowerShell commands that were executed. The threat group used multiple tools to help with reconnaissance, lateral movement across a network, remote service installation, and remote sessions management, by leveraging living-off-the-land techniques (i.e. utilities and platforms that are already available on a system or device), off-the-shelf hacking tools, and custom-built backdoors. Some of the tools leveraged were "the Non-Sucking Service Manager" for ensuring their critical components, such as the proxy or the RAT, are up and running. The group also uses Sysinternals tools such as 'psexec', for lateral movement within a network, along with tools such as Mimikatz and SafetyKatz. The threat group created a separate user account on compromised devices to perform different actions, such as using the credentials with 'psexec' tool to gain access to specific devices. Upon inspection of multiple compromised devices, researchers found shellcode from the Metasploit framework that was used to connect to different internal IP addresses.

 

Researchers believe that the initial infection vector in the Kuwait campaign was a malicious document sent through spearphishing emails. This was possibly used to secure a foothold inside the network and then the threat group installed custom modules, such as a modified PLINK tool (part of a module from the freely available PUTTY tool). The tool was customized to add new features allowing the tool to be run as a Windows service. The threat group also installed a custom backdoor that could be executed as a service. They used credentials to create a separate user account on the device that was used to execute actions inside the network. Uniquely, a command and control (C2) address was passed as a command line argument to both the proxy tools and the backdoor.

 

Researchers identified that for the Saudi Arabia campaign, the threat actors used social engineering tactics to trick the user into running a remote administration tool (RAT) and found evidence of internal reconnaissance but state that is was not as extensive as the one conducted on Kuwait companies. There was also no evidence of lateral movement within this campaign. Researchers believe this was because the threat actors were not able to find any vulnerable machines.

Recommendations

Deloitte CTI recommends users

·        To perform scan for IOCs and block them.

·        Threat actors are using phishing campaign as the initial attack vector with malicious attachment. Using Group Policy Object (GPO) based corporate email controls to block users from enabling macros in Microsoft Office attachments typically used for malware downloaders/droppers - including both Word documents and Excel spreadsheets - originating from outside your network.

·        The tools used by the threat actors such as Sysinternals tools-'psexec', for lateral movement within a network, Mimikatz and SafetyKatz for credential dumping, Metasploit shellcode is used for the reverse shell. Hence the monitoring teams should observe and do the analysis of any activity with these tools.
