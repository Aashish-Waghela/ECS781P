Threat Notification

Updated version of Amadey loader introduces functions to capture screenshots, steal credentials and deliver Remcos RAT

Malware

Type: Incident | Issue Date : 27-05-2020 | TLP: Green | Industry: All | Region: All | Serial: G-TN-EN-01-13023

Summary

On May 20, 2020, security researchers at Zscaler reported on an updated version of the Amadey loader. Researchers initially discovered Amadey in late 2018, when it was sold on a Russian-speaking cybercriminal forum. Amadey infects a victim's machine and incorporates the infected machine into a botnet. Threat actor(s) primarily use Amadey for collecting information on a victim's environment, which it then sent back to a command-and-control (C2) server. It also delivers additional malware such as Remcos RAT. Recently, the Amadey botnet was observed being used by the financially-motivated cybercriminal threat group TA505 to deliver spam emails containing the 'FlawedAmmyy' remote access Trojan (RAT). In January 2020, security researchers at Cylance reported on the earlier Amadey version (v1.09) delivered via the 'AZORult' infostealer.

 

According to the report, researchers at Zscaler spotted an updated version of Amadey being delivered via RIG Exploit Kit. This latest version has an updated control panel login page "a2020 AMADEY" and includes functions such as screen capturing and installation of Remcos RAT. The malware is written C++ programming language and is packed with a crypter. Upon execution, Amadey first looks for antivirus (AV) products installed on the victim machine by using the function '_Z8aCheckAVv()'. If no AV software is found to be installed, Amadey then copies itself under "%ProgramData%" directory and adds a key to the Windows Startup Registry to maintain persistence. Unlike the 1.09 version, the updated Amadey version then attempts to download and load two DLL files named "cred.dll" and "scr.dll" by using the 'LoadPluginPc()' function onto the victim machine. The file "cred.dll" is responsible for stealing credentials from applications such as FileZilla, Pidgin, WinSCP, TigerVNC, RealVNC, TightVN installed on the victim's machine. The file "scr.dll" is responsible for capturing system screenshots. The harvested credentials and screenshots, along with system information are then exfiltrated to the C2 server via HTTP POST requests. While analyzing the Amadey control panel, researchers observed the majority of the infected machines located in Canada. Additionally, threat actor(s) were also observed to be actively distributing the Remcos RAT via the control panel across all Amadey controlled bots (i.e. infected machines).

Recommendations

Deloitte CTI recommends users to:

·        Monitor network for given IOCs and block them.

·        Keep their anti-malware and anti-spyware software up-to-date with the latest packages.

·        For employees, alert the IT security team of potentially suspicious emails and files.

·        Apply the latest software patches from OS and third-party vendors.

·        Exercise good email and website safety practices.
